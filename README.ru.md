## ✨ Возможности

### Серверы

- Запускайте сколько угодно серверов параллельно: Vanilla, Paper, Spigot, Purpur, Bukkit, Fabric, Forge, Velocity или свой jar
- Запуск, остановка, перезапуск и ввод команд в консоль, вывод отображается в реальном времени через WebSocket
- Сервер можно продублировать, выгрузить в один файл или, наоборот, импортировать. Серверы раскладываются по папкам, а лишние удаляются пачкой
- Новый сервер создаётся через короткий wizard, а подходящая версия Java скачивается сама — не нужно искать нужный JRE вручную

### Плагины и моды

- Поиск и установка плагинов (Bukkit/Spigot/Paper) и Fabric-модов из Modrinth
- Если что-то устарело, обновление — в один клик
- Зависимости подтягиваются автоматически

### Мониторинг и диагностика

- CPU, RAM и онлайн по каждому серверу в реальном времени, плюс ресурсы самого хоста (CPU, память, диски)
- Графики истории CPU/RAM — можно посмотреть, что было, а не только текущий момент
- Список игроков с никами, аватарами и статусом онлайн — читается по протоколу Query
- Анализ логов ловит типовые проблемы — нехватку памяти, занятый порт, повреждённый мир, краши плагинов и модов, не ту версию Java — а панель диагностики там, где может, предлагает решение

### Файлы, консоль и бэкапы

- Файловый менеджер с редактором Monaco (тот самый, что в VS Code), загрузкой перетаскиванием и работой с архивами
- Real-time консоль с вводом команд, умными подсказками + отдельный интерфейс для просмотра старых логов
- Создание и восстановление бэкапов: отдельных файлов или сервера целиком

### Автоматизация

- **Планировщик** — задачи по cron, по простому интервалу или однократно в заданное время. Действие — это запуск/остановка/перезапуск, команда в консоль, бэкап или исходящий вебхук. Есть превью cron-расписания и история запусков по каждой задаче, чтобы видеть, что реально сработало
- **Telegram-бот** — уведомления и удалённое управление: вход по OTP, кнопки управления серверами и даже мастер создания сервера, на русском или английском

### Доступ и права

- Несколько аккаунтов с правами по каждой функции (см. [Роли и права](#-роли-и-права-rbac) ниже) и опциональным доступом к конкретным серверам
- Двухфакторка — приложение TOTP или подтверждение нажатием в Telegram — и список активных сессий, каждую из которых можно отозвать
- Журнал аудита: кто что сделал, с поиском и фильтрами

### Под себя

- Дашборд, который можно собрать как удобно: виджеты с серверами, ресурсами хоста, диагностикой и бэкапами перетаскиваются мышкой
- Командная палитра позволяет быстро переключаться между вкладками с клавиатуры
- Типы серверов описываются **blueprint`ами**, а **система расширений** панели умеет добавлять новые функции, UI, обработчики событий и собственные права
- Встроенный FTP-сервер
- Полная локализация на русский и английский

## 🔐 Роли и права (RBAC)

Каждому аккаунту выдаются ровно те права, которые ему нужны, и не больше

| Право                | Ключ                | Что открывает                                  |
| -------------------- | ------------------- | ---------------------------------------------- |
| Просмотр серверов    | `servers_view`      | Видеть серверы и их состояние                  |
| Управление серверами | `servers_control`   | Запуск / остановка / перезапуск, консоль       |
| Настройка серверов   | `servers_configure` | Править настройки и `server.properties`        |
| Создание серверов    | `servers_create`    | Создавать, дублировать и импортировать серверы |
| Плагины и моды       | `plugins_mgr`       | Ставить и обновлять плагины и моды             |
| Файловый менеджер    | `file_manager`      | Чтение и запись файлов сервера                 |
| Управление Java      | `java_mgr`          | Скачивать и удалять версии Java                |
| Бэкапы               | `backups`           | Создавать и восстанавливать бэкапы             |
| Планировщик          | `scheduler_mgr`     | Создавать и вести задачи планировщика          |
| Мониторинг системы   | `system_monitoring` | Смотреть CPU / RAM / диски хоста               |
| Журнал аудита        | `audit_log`         | Читать журнал аудита                           |
| Аккаунты             | `accounts_mgr`      | Создавать и редактировать другие аккаунты      |
| Настройки панели     | `kubek_settings`    | Менять общие настройки панели                  |

Сверх того, аккаунт без прав администратора можно **ограничить конкретными серверами** — удобно, когда нужно выдать человеку доступ к одному серверу, не показывая остальные. А поскольку права — это просто реестр, **расширения могут заводить свои собственные**: они появляются в редакторе аккаунта рядом со встроенными

## 🧩 Расширения и blueprint`ы (для разработчиков)

Kubek рассчитан на расширение извне, без форка. **Blueprint`ы** описывают типы серверов — как скачивается, настраивается и запускается jar, — а **расширения** добавляют целые функции: новые страницы и UI, маршруты API, обработчики событий и собственные права

Два SDK со стартовыми шаблонами помогут начать за пару минут:

| SDK                                                                            | npm                                                                                | Стартовый шаблон                                                          |
| ------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| **Extension SDK** — функции, UI, маршруты API, события и права                  | [`@kubekpanel/extension-sdk`](https://www.npmjs.com/package/@kubekpanel/extension-sdk) | [KubekPanel/extension-template](https://github.com/KubekPanel/extension-template) |
| **Blueprint SDK** — описание новых типов серверов (скачивание, конфиг, запуск)  | [`@kubekpanel/blueprint-sdk`](https://www.npmjs.com/package/@kubekpanel/blueprint-sdk) | [KubekPanel/blueprint-template](https://github.com/KubekPanel/blueprint-template) |

## 🚀 Быстрый старт

### Установка одной командой (Linux / macOS)

```bash
curl -fsSL https://raw.githubusercontent.com/Seeroy/kubek-minecraft-dashboard/main/install.sh | bash
```

Скрипт определит систему и сам либо скачает нативный бинарник, либо развернёт Docker. Затем откройте **http://localhost:8000**.

<details>
<summary>Параметры скрипта установки</summary>

```bash
# Принудительно через Docker Compose
curl -fsSL .../install.sh | bash -s -- --docker

# Принудительно нативный бинарник (ошибка, если платформа не поддерживается)
curl -fsSL .../install.sh | bash -s -- --binary

# Своя папка / версия / установка без запуска
curl -fsSL .../install.sh | bash -s -- --dir /opt/kubek --version v4.0.0 --no-run
```

</details>

### Docker Compose (сборка из исходников)

```bash
git clone https://github.com/Seeroy/kubek-minecraft-dashboard.git
cd kubek-minecraft-dashboard
docker compose up -d
```

### Docker (готовый образ)

```bash
docker run -d --name kubek -p 8000:8000 -v kubek-data:/data seeroy/kubek-minecraft-dashboard:latest
```

### Готовый бинарник

Скачайте бинарник под свою ОС со [страницы релизов](https://github.com/Seeroy/kubek-minecraft-dashboard/releases) и запустите:

```bash
chmod +x Kubek-*-linux-x64
./Kubek-*-linux-x64
```

## 🔄 Миграция с предыдущей версии (1.x - 3.x)

Переходите со старой версии Kubek? Миграция **автоматическая** — Kubek сам импортирует старые данные при первом запуске, отдельный инструмент не нужен

1. Положите файлы старой установки в рабочую папку Kubek (`/data` в Docker):
   - `config.json` — настройки панели (язык, FTP, Telegram-бот, порт веб-сервера…)
   - `users.json` — аккаунты и их права
   - `servers/` — папку целиком, вместе с `servers/servers.json` и файлами серверов
2. Запустите Kubek и **следите за выводом в консоли**

Что произойдёт:

- ⚙️ **Настройки** перенесутся в новую базу
- 🎮 **Серверы** импортируются, а их папки переименуются в новый формат `servers/<uuid>`. Jar-файл ядра и флаги JVM (`-Xmx`/`-Xms`) определяются из старых стартовых скриптов автоматически
- 👥 **Аккаунты** переносятся вместе с правами, но **пароли генерируются заново**. Они выводятся в консоль один раз — сразу их сохраните

После успешного импорта старые файлы переименуются в `*.migrated-backup`, и миграция больше не запустится. Если что-то пошло не так — остановите Kubek, верните файлы на место и запустите заново.

> Совет: на всякий случай сделайте копию старых данных перед миграцией :)

## ⚙️ Настройка

| Переменная | По умолчанию | Описание                                    |
| ---------- | ------------ | ------------------------------------------- |
| `PORT`     | `8000`       | Порт веб-панели (HTTP + WebSocket)          |
| `FTP_PORT` | `21`         | Порт встроенного FTP-сервера (если включён) |
| `TZ`       | `UTC`        | Часовой пояс контейнера                     |

При первом запуске мастер настройки создаст учётную запись администратора

### Данные и их сохранность

Всё хранится в рабочей директории (`/data` в Docker) — смонтируйте её как том, чтобы данные не потерялись:

| Путь             | Содержимое                          |
| ---------------- | ----------------------------------- |
| `db.sql`         | База панели (конфиг, пользователи…) |
| `servers/`       | Все файлы Minecraft-серверов        |
| `backups/`       | Архивы бэкапов                      |
| `binaries/java/` | Автоматически скачанные версии Java |

### Порты и Minecraft-серверы

Каждый Minecraft-сервер использует свой порт (`25565` по умолчанию). Пробросьте каждый в `docker-compose.yml`, либо на Linux используйте `network_mode: host`, чтобы открыть их все сразу

## 🛠️ Сборка из исходников

Нужен [Bun](https://bun.sh)

```bash
git clone https://github.com/Seeroy/kubek-minecraft-dashboard.git
cd kubek-minecraft-dashboard

bun install
(cd frontend && bun install)
(cd backend && bun install)

bun run build                   # все платформы → dist/
bun run build --platform linux  # только одна платформа
bun run build --platform native # текущая архитектура (для Docker)
```

Либо соберите Docker-образ сами:

```bash
docker build -t kubek .
docker buildx build --platform linux/amd64,linux/arm64 -t kubek .
```
